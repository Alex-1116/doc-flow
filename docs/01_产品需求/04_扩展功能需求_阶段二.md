# 扩展功能需求 (Phase 2) - Extended Features

基于第一阶段构建的稳定基石（文件解析、结构化提取、多文件对比、摘要与 QA），**阶段二**的目标是深入特定垂直场景，提供差异化价值，将 **DocFlow** 从“文档工具”进化为“场景化智能助手”。

## 1. 代码解释与代码库问答 (Code Explanation & Repository QA)
**目标：** 解决开发者接手新项目、理解复杂代码片段的痛点。
- **需求描述**：
  - **支持代码文件解析**：原生支持 `.py`, `.ts`, `.tsx`, `.js`, `.go`, `.java`, `.cpp` 等常见代码文件的上传和解析。
  - **基于代码片段解释**：用户在聊天框粘贴一段代码，AI 能结合已入库的架构文档或代码文件，给出详细的执行逻辑解释。
  - **全仓搜索与问答**：允许上传打包的 ZIP 源码包，AI 能回答“项目中的鉴权逻辑是如何实现的？”，“某个接口在哪个文件里？”。
- **技术实现路径**：
  - 引入 `LanguageParser` 或定制代码切片器 (AST-based Splitter) 以保留代码函数或类的完整上下文。
  - 对代码增加特定的 Metadata（语言类型、文件路径、函数名）。

## 2. 自动化 PR 助手 (Automated PR Assistant)
**目标：** 减轻代码审查负担，自动生成审查意见或 PR 描述。
- **需求描述**：
  - **代码变更解读**：上传 `.patch` 或 `.diff` 文件，AI 自动总结本次 PR 做了哪些变更。
  - **风险与建议生成**：针对修改的代码，自动指出潜在的风险点（如边界条件未处理、性能问题），并提供 Code Review 意见。
  - **生成 Release Notes**：基于一堆 PR 描述自动归纳出版本发布说明。
- **技术实现路径**：
  - 解析 Diff 格式文件。
  - 使用针对代码审查优化的 Prompt，强制输出结构化 Review 数据。

## 3. 会议纪要智能处理 (Intelligent Meeting Minutes)
**目标：** 自动处理会议录音文本，极大提升参会人员后续跟进的效率。
- **需求描述**：
  - **录音转写文本接入**：支持上传飞书、钉钉、Zoom 等工具导出的长篇会议录音转写文本。
  - **智能议题分割**：自动将杂乱无章的对话记录，按时间线或话题分割成多个议题模块。
  - **自动提取 Action Items**：提取出每个人的待办事项及截止日期。
- **技术实现路径**：
  - 针对对话体文本（Speaker: Content 格式）优化分块策略，避免将同一个人的连贯发言切断。
  - 提供定制化的会议纪要 Prompt 模板。

## 4. 浏览器辅助插件 (Browser Extension Integration)
**目标：** 将 DocFlow 的能力延伸到用户日常浏览网页的场景中，无需手动下载文件再上传。
- **需求描述**：
  - **网页正文提取与问答**：在 Chrome/Edge 插件中一键提取当前网页正文，直接发送给本地的 DocFlow 后端进行临时入库和问答。
  - **划词解释**：在网页中选中文本，右键调用 DocFlow 解释该段落的含义（如晦涩的技术名词或论文段落）。
  - **网页自动摘要**：打开长篇文章后，插件自动在侧边栏生成当前页面的摘要。
- **技术实现路径**：
  - 开发 Chrome Extension，提供与 DocFlow 后端的鉴权与通信。
  - 后端提供一个临时的 `Temporary Collection`，专门存放网页片段，定时清理，避免污染正式的文档知识库。

## 5. 企业级部署与协作 (Enterprise Deployment & Collaboration)
**目标：** 从单机版演进为支持小型团队共享的内部知识平台。
- **需求描述**：
  - **多用户与权限隔离**：支持简单的账号体系，每个人有自己私有的知识库，同时可以共享给团队。
  - **全局知识库搜索**：支持在团队公共的规范、API 文档中进行全局语义检索。
  - **SSO 与飞书/钉钉集成**：支持企业微信、飞书扫码登录。
- **技术实现路径**：
  - 引入 PostgreSQL 或 MySQL 管理用户与文档的元数据关联关系。
  - ChromaDB 的 Collection 按 `user_id` 或 `tenant_id` 划分，或者在查询时强制附带 Metadata Filter。
