# 📄 DocFlow - 智能文档摘要与问答系统

> **让 AI 帮你读文档。本地优先，隐私友好。**

---

## 🎯 项目概述

### 背景
开发者每天需要阅读大量的技术文档、PR 说明、API 文档、会议记录，但：
- ⏱️ **时间不够**：一篇 50 页文档读完需要 2 小时
- 🧠 **记不住**：读完就忘，需要反复查阅
- 🔍 **找不着**：想找某个细节时需要全文搜索

### 目标
构建一个 **本地优先、隐私友好** 的智能文档助手，让 AI 帮你读文档。

---

## ✨ 核心功能

| 模块 | 功能 | 价值 |
|------|------|------|
| 📄 **文档解析** | 支持 PDF、Markdown、Word、代码文件 | 10+ 格式支持 |
| 📝 **智能摘要** | 提取关键信息，30秒读完长文档 | 节省 90% 时间 |
| 💬 **文档问答** | 对着文档提问，AI 精准回答 | 不用全文搜索 |
| 📊 **要点提取** | Action Items / 决策 / 风险自动标红 | 不会遗漏重要信息 |
| 🔗 **语义搜索** | 按意思查找，不是按关键词 | 搜索准确率 ×3 |
| 🧩 **多文档对比** | 对比多个版本的文档差异 | PR 审查神器 |

## 💡 技术选型说明

### ✅ 为什么全栈本地运行？

1. **隐私第一** - 文档不需要上传到 OpenAI，公司机密完全安全
2. **零 API 费用** - 完全免费，没有 token 计费
3. **开源可控** - 所有组件都是开源的
4. **硬件友好** - M1/M2/M3 Mac 可以流畅运行 7B 模型
5. **企业级** - 直接可以用于公司内部部署

### ❌ 为什么不选 OpenAI API？

- 隐私问题：公司机密文档不能外传
- 成本问题：1000 个文档 ≈ $100+
- 网络延迟：比本地慢 5-10 倍
